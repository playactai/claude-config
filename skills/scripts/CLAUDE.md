# scripts/

Python package root for skill implementations. uv-managed; `pyproject.toml` declares the package editable-installable and pins dev deps (pytest, hypothesis, ruff, pyright).

## Files

| File                       | What                                                           | When to read                                                       |
| -------------------------- | -------------------------------------------------------------- | ------------------------------------------------------------------ |
| `pyproject.toml`           | uv deps, ruff/pyright/pytest config, hatch wheel target        | Adding deps, tuning lint/type rules, changing packaged modules     |
| `uv.lock`                  | Pinned dependency lockfile (generated by `uv lock`)            | Never edit directly; regenerate via `uv lock` after dep changes    |
| `validate_conventions.py`  | CI check — `get_convention()` calls vs `conventions/REGISTRY.yaml` | Debugging convention-access violations in skill scripts          |

## Subdirectories

| Directory  | What                                             | When to read                                              |
| ---------- | ------------------------------------------------ | --------------------------------------------------------- |
| `skills/`  | Python package — all skill implementations       | Executing or editing any skill                            |
| `tests/`   | Pytest suite (workflow import/structure/steps, AST, domain types) | Running tests, adding property-based tests           |

## Test

```bash
SCRIPTS="${CLAUDE_PROJECT_DIR:-$HOME}/.claude/skills/scripts"
uv run --project "$SCRIPTS" pytest "$SCRIPTS" -v
```

## Lint / Type-check

```bash
SCRIPTS="${CLAUDE_PROJECT_DIR:-$HOME}/.claude/skills/scripts"
uv run --project "$SCRIPTS" ruff check "$SCRIPTS"
uv run --project "$SCRIPTS" pyright "$SCRIPTS"
```
